Project Overview

This project is a detailed study of an open-source software system, focusing on its origin, license, working, and impact on the software industry.

For this project, I have chosen Git, a widely used distributed version control system. The objective of this project is to understand how Git works, why it was created, and how it supports collaborative software development.

The project also includes practical implementation of Linux shell scripting to demonstrate command-line skills and system-level understanding.

About Git

Git is a distributed version control system developed by Linus Torvalds in 2005. It helps developers track changes in code, collaborate efficiently, and maintain project history.

Key features of Git:

Distributed system (no central server required)
Fast and efficient
Supports branching and merging
Widely used in modern software development

Shell Scripts Description
Script 1 — System Identity Report

Displays system information such as kernel version, user details, uptime, date, and Linux distribution.

Script 2 — FOSS Package Inspector

Checks whether Git is installed on the system and displays its version and details.

Script 3 — Disk and Permission Auditor

Analyzes important system directories and displays their size, permissions, and ownership.

Script 4 — Log File Analyzer

Reads a log file and counts occurrences of a specific keyword such as "error".

Script 5 — Open Source Manifesto Generator

Generates a personalized open-source philosophy statement based on user input.
